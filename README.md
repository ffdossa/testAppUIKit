Додаток для перегляду стрічки постів та їх деталей.
Архітектура та інструменти
* Архітектура: MVVM (Model-View-ViewModel). Вся бізнес-логіка та стан розширення комірок винесені у ViewModel. View лише відображає дані.
* UI: UIKit, повністю програмна верстка (Code-only).
* Layout: Auto Layout за допомогою NSLayoutConstraint. Інтерфейс адаптивний до ширини екрана.
* List: UITableView з використанням класичного UITableViewDataSource.
* Network: URLSession + Codable з використанням сучасного async/await.

Рішення "Розгорнути/Згорнути"
Стан "розгорнутості" (expanded) не зберігається в комірці, а винесений у Set<Int> у FeedViewModel для коректної роботи при скролі (reuse cells).
1. Розрахунок: У PostCell ми перевіряємо, чи текст перевищує ліміт, використовуючи boundingRect для поточного шрифту та ширини контейнера.
2. Відображення: Якщо текст довгий (> 2 рядків), з'являється кнопка "Розгорнути", а numberOfLinesвстановлюється у 2.
3. Дія: При натисканні кнопки ViewModel оновлює Set (додає або видаляє ID) і повідомляє контролер.
4. Оновлення: Контролер викликає tableView.beginUpdates() та endUpdates() (або reloadRows), що змушує таблицю перерахувати висоту конкретної комірки з анімацією без повного перезавантаження даних.

Особливості реалізації
* Зображення: Асинхронне завантаження зображень з кешуванням URLSession (у спрощеному вигляді для тестового).
* Дати: Конвертація Unix Timestamp у читабельний формат DateFormatter.
* Обробка помилок: Централізована обробка помилок декодування JSON (включно з keyNotFound для відлагодження API).
